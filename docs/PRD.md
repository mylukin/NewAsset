# 二、产品概述

## 2.1 背景

当前物业/资产运营过程中，房源、住宅配套资产、车库资产、场馆资产、办公资产等数据分散在 Excel、纸质台账或不同系统中，缺乏统一的资产编码与资产全生命周期管理，导致：

* 资产信息不统一，难以准确盘点与对账；
* 维护记录与资产脱节，无法真实反映资产状态；
* 经营指标（如出租率、利用率、完好率）难以快速获取。

本项目建设一套“资产管理系统”MVP，聚焦 6 个业务模块，通过统一资产编码和资产台账，实现基础资产管理与维护管理，为后续经营分析和扩展功能打基础。

## 2.2 建设目标

* 建立统一的资产编码规则和资产台账；
* 管理房源、住宅配套、车库、场馆、办公等五类资产；
* 管理资产维护工单与维保计划，并与资产状态联动；
* 提供基础统计视图（空置率、利用率、报修量等）；
* 支持按项目维度进行权限控制和数据查看。

## 2.3 范围说明（MVP）

纳入范围（6 个业务模块）：

1. 资产维护模块
2. 房源资产模块
3. 住宅配套资产模块
4. 车库资产模块
5. 场馆资产模块
6. 办公资产模块

通用能力依托现有平台（如 RuoYi-Vue）提供：用户/角色、部门/项目、菜单权限、数据权限、字典管理、登录日志、操作日志等，PRD 不再展开。

# 三、用户角色与权限

## 3.1 角色定义

1. 系统管理员

   * 管理用户、角色、菜单权限与数据权限；
   * 管理基础字典（资产类别、状态、项目等）；
   * 管理资产编码规则。

2. 资产管理员

   * 负责各类资产（房源/配套/车库/场馆/办公）的录入、变更、审核；
   * 维护资产基础信息和状态；
   * 发起/查看与资产相关的维护工单。

3. 运维/维修人员

   * 接收和处理资产维护工单；
   * 更新工单处理结果和资产状态；
   * 填写日常巡检/保养记录。

4. 业务运营/招商人员

   * 关注房源、车库、场馆等资产的可用量、出租率、利用率；
   * 查看基础经营指标和资产列表；
   * 不可修改资产关键信息。

5. 管理层

   * 通过首页/统计视图查看关键指标（资产数量、空置率、报修量等）；
   * 只读权限。

## 3.2 权限控制

* 菜单/按钮权限：按角色配置；
* 数据权限：按项目/部门维度控制，典型方式：

  * 资产管理员：可管理授权项目下的资产；
  * 运维人员：可查看和处理分派给自己的工单及相关资产；
  * 运营/管理层：按项目范围只读查看。

# 四、统一概念与通用需求

## 4.1 项目与组织

* 系统支持多项目管理，每条资产记录必须关联一个“项目”；
* 项目、部门等组织结构交由平台通用模块维护。

## 4.2 统一资产编码规则（重点）

### 4.2.1 目标

* 为所有资产生成全局唯一的“资产编码”；
* 资产编码自动生成，尽量减少人工输入，避免重复与错误；
* 支持不同资产类别使用不同的编码模板，但遵循统一规范。

### 4.2.2 规则要求

1. 统一规则组成示例（可配置）：

   * 项目代码（必选）
   * 资产类别代码（必选）
   * 位置编码（如楼栋号/区域号/车场编号）
   * 楼层（可选）
   * 顺序号（必选，固定长度自动递增）

2. 配置方式（系统管理员使用）：

   * 支持按“资产类别”配置编码模板：

     * 示意：`{项目代码}-{类别代码}-{楼栋}-{楼层}-{四位流水}`
   * 可配置每一段的长度、补零规则；
   * 可预览生成结果，确保无冲突。

3. 编码生成方式：

   * 新增资产时，系统根据选择的项目、类别、位置等，自动生成编码；
   * 特殊情况允许人工修改，但需权限 + 日志记录；
   * 不允许重复编码。

4. 编码使用要求：

   * 所有业务页面优先展示资产编码；
   * 维护工单、统计报表均使用资产编码作为主标识之一。

## 4.3 通用资产字段

所有资产模块基础字段（建议）：

* 资产编码（唯一，系统生成）
* 资产名称
* 资产类别（字典：房源/配套/车位/场馆/办公等）
* 所属项目
* 所在位置：

  * 楼栋/区域
  * 楼层
  * 房间号/车位号/场馆号等
* 权属类型（自有/租入/托管等）
* 权属单位/业主
* 使用部门/责任部门
* 责任人
* 使用状态（统一状态机中的状态，见 4.4）
* 购置日期/投用日期
* 原值
* 折旧方式（字典：平均年限法等）
* 已提折旧额
* 资产净值（可计算）
* 品牌型号
* 供应商
* 保修到期日
* 存放位置描述（自由文本）
* 备注
* 附件（合同、照片、图纸等）

## 4.4 统一资产状态机

统一抽象一套状态（可由字典维护）：

* 在建/未投入使用
* 可用-自用
* 可用-对外出租
* 可用-空置
* 暂停使用（如装修）
* 故障/停用
* 维修中
* 计划报废
* 已报废

要求：

* 各模块具体的业务状态通过映射归到上述统一状态；
* 维护工单状态变化时，可触发资产状态联动（例如“维修工单已完成”→ 资产状态从“维修中”变回“可用-自用/对外出租”等）。

## 4.5 附件与图片

* 所有资产与维护工单支持上传附件（照片、合同扫描件、验收单等）；
* 支持多文件上传、预览与下载。

## 4.6 操作日志与审计

* 对资产关键字段修改记录日志（修改前后值、操作人、时间）；
* 对维护工单的状态流转记录日志；
* 日志可在详情页查看。

# 五、核心业务流程

## 5.1 资产新增与编码流程

1. 资产管理员选择模块（房源/车库/办公等）→ 点击“新增资产”；
2. 填写所属项目、位置、资产类别等基础信息；
3. 系统根据编码规则生成资产编码并展示；
4. 用户补充其他信息（原值、权属、责任人等）；
5. 提交保存；
6. 如需审核，可配置为“草稿 → 待审核 → 已生效”流程。

## 5.2 资产维护流程（报修工单）

1. 资产管理员/运营人员/运维人员在资产详情或列表中点击“发起维护”；
2. 填写故障描述、紧急程度、要求完成时间等，提交工单；
3. 系统自动或由管理员派单给具体运维人员；
4. 运维人员接单，记录处理过程、耗材、工时；
5. 运维处理完成，提交结果（建议附照片）；
6. 发起人/资产管理员进行验收确认，工单关闭；
7. 工单结果如涉及资产状态变更（如“修复完成”），自动更新资产状态。

## 5.3 定期巡检/保养流程（可作为 P1）

1. 设置维保计划（资产类别/项目/周期等）；
2. 到期系统自动生成维护工单；
3. 运维人员按计划执行、填写结果；
4. 统计维保完成率、缺陷情况。

## 6.1 资产维护模块

### 6.1.1 模块定位

统一管理所有与资产相关的维护活动，包括报修、维保、巡检等，形成可追溯的维护记录，并与资产状态联动。

### 6.1.2 功能点

1. 维护工单管理（P0）

   * 列表

     * 按项目、资产类别、工单状态、紧急程度、责任人等过滤；
     * 支持按创建时间、计划完成时间排序；
   * 新建工单

     * 支持从“资产详情页”一键发起，自动带出资产编码、名称、位置等；
     * 支持从“维护模块”独立发起工单，并选择关联资产；
     * 字段：

       * 工单编号（系统自动生成）
       * 关联资产（必填，支持搜索选择）
       * 项目（自动带出资产项目，可修改权限控制）
       * 故障类型（字典）
       * 故障/需求描述（文本）
       * 紧急程度（一般/重要/紧急）
       * 要求完成时间
       * 发起人（自动）
       * 附件（照片等）
   * 派单

     * 指派责任人（运维人员），可按项目/区域过滤人员；
     * 派单时间记录；
   * 处理过程

     * 运维人员可更新处理进度：

       * 接单时间
       * 处理描述
       * 耗材与费用（可选）
       * 实际完成时间
       * 附件（维修前后照片等）
   * 验收与关闭

     * 验收人（默认发起人/资产管理员）
     * 验收结果（通过/不通过）
     * 不通过时可退回继续处理；
     * 通过后工单状态为“已关闭”。
   * 状态流转

     * 状态字典：待派单、待接单、处理中、待验收、已关闭、已取消；
     * 状态流转记录（带操作人和时间）。

2. 维护记录查询（P0）

   * 在资产详情页展示该资产所有历史工单列表；
   * 支持按时间、类型过滤；
   * 支持导出 Excel。

3. 资产状态联动（P0）

   * 发起工单时可选择是否将资产状态改为“故障/维修中”；
   * 工单关闭时按配置规则恢复资产状态（如恢复为原状态，或人工选择）。

4. 维保计划（P1）

   * 按资产类别、项目、周期（每月/每季度/每年）配置维保计划；
   * 到期自动生成工单；
   * 支持查看计划执行情况。

## 6.2 房源资产模块

### 6.2.1 模块定位

管理全部房源资产的全生命周期信息，包括基础属性、权属、使用状态、租赁情况、维护记录等。

### 6.2.2 功能点

1. 房源资产列表（P0）

   * 搜索与筛选：

     * 项目、楼栋、楼层、房号；
     * 权属类型、使用状态（自用/出租/空置等）；
     * 面积区间；
   * 支持导出 Excel；
   * 支持批量更新部分字段（如责任人、使用部门）。

2. 新增/编辑房源资产（P0）

   * 字段包含：

     * 通用资产字段（见 4.3）
     * 房源特有字段：

       * 房号
       * 建筑面积
       * 套内面积
       * 户型（字典：一居/两居等）
       * 房屋用途（住宅/商用/公寓等）
       * 当前用途（自住/出租/空置等）
       * 当前使用人/承租人（仅展示基本信息）
       * 合同编号（外部系统或手工录入）
       * 租金单价/租金总额（选填）
   * 支持复制已有房源（同楼层多户场景）。

3. 房源详情页（P0）

   * 展示：

     * 基础信息
     * 权属与使用信息
     * 关联维护记录（来自维护模块）
     * 关联合同信息（仅记录编号等，不做合同系统）
   * 支持在详情页快速发起维护工单。

4. 房源统计视图（P0）

   * 统计指标：

     * 总房源数量
     * 自用数量/占比
     * 出租数量/占比
     * 空置数量/占比
   * 按项目维度展示，可切换项目。

## 6.3 住宅配套资产模块

### 6.3.1 模块定位

管理附属于住宅的固定设施，如公共区域设备、家具家电等，支持独立建档和维护。

### 6.3.2 功能点

1. 配套资产列表（P0）

   * 筛选条件：

     * 项目、楼栋、楼层；
     * 资产类型（如电梯、门禁、健身设施等）；
     * 使用状态；
   * 支持 Excel 导出。

2. 新增/编辑配套资产（P0）

   * 除通用资产字段外，建议增加：

     * 配套类型（字典，如电梯/门禁/公共照明等）
     * 安装位置描述（如“1 号楼 1 单元大堂西侧”）
     * 设备编号（原厂家编号）
     * 维保单位
     * 维保电话

3. 详情与维护记录（P0）

   * 展示资产详情、维护记录列表；
   * 支持一键发起维护工单。

4. 维保到期提醒（P1）

   * 根据保修到期日或维保周期，对即将到期的配套资产给出提醒列表。

## 6.4 车库资产模块

### 6.4.1 模块定位

管理车位等车库资产，包括自用/出租/空置状态及使用情况。

### 6.4.2 功能点

1. 车位列表（P0）

   * 筛选条件：

     * 项目、车库区域（地上/地下/编号）；
     * 车位号；
     * 使用状态（自用/出租/空置/外租等）；

2. 新增/编辑车位资产（P0）

   * 特有字段：

     * 车位类型（固定/临停车位等）

     * 当前使用人/车牌号（可选）
     * 租金单价/包月金额（可选）

3. 车位详情（P0）

4. 车位利用率统计（P0）

   * 指标：

     * 总车位数
     * 自用车位数/占比
     * 出租车位数/占比
     * 空置车位数/占比
   * 按项目/车库区域展示。

## 6.5 场馆资产模块

### 6.5.1 模块定位

管理场馆类资产（如活动室、体育场地、会议室等），关注开放情况与使用情况。

### 6.5.2 功能点

1. 场馆列表（P0）

   * 筛选条件：

     * 项目、场馆类型（篮球场、羽毛球馆、会议室等）、使用状态；

2. 新增/编辑场馆资产（P0）

   * 特有字段：

     * 场馆编号/名称
     * 场馆类型
     * 容纳人数
     * 使用方式（免费/收费/内部使用等）
     * 对外开放时间段描述（文本）

4. 使用情况统计（P1）

   * 先以手动记录的“使用记录（日期、时段、用途）”为基础进行统计；
   * 后续可对接预约系统。

# 七、首页与统计视图（跨模块）

## 7.1 首页概览（P0）

按角色展示不同内容：

1. 管理层/运营角色首页

   * 指标卡片：

     * 总资产数量
     * 房源空置率
     * 车位空置率
     * 本月新建维护工单数
     * 当前未关闭维护工单数
   * 最近维护工单列表（Top N）；
   * 可切换项目。

2. 运维角色首页

   * “待处理工单”列表（默认按紧急程度排序）；
   * 我的本月工单数量统计（已完成/进行中）。

3. 资产管理员首页

   * “待完善资产信息”列表（如缺少关键字段的资产）；
   * “最近新增资产”列表；
   * 资产数量统计图（按类别）。

# 八、非功能需求（简要）

1. 性能

   * 每个列表页采用分页查询，单页默认 20 条记录；
   * 在典型数据量 10 万条资产、10 万条工单下，常规查询响应时间 ≤ 3 秒（合理索引和分页条件下）。

2. 安全

   * 采用平台现有登录认证与授权机制；
   * 接口只对登录用户开放，敏感操作（删除/状态变更）需要相应权限。

3. 审计

   * 关键字段变更有记录；
   * 删除采用软删除方式（保留数据以便追溯）。

4. 可用性

   * PC Web 为主，适配 1920×1080 分辨率；
   * 表格列支持显示/隐藏配置、列宽拖拽（可作为 P1）。
